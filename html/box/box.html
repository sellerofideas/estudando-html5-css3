Entendendo as boxes

Todo conteúdo é criado em forma de em caixas, encergar a pagina de acordo com essa perpectiva ajuda e muito na hora de criar. Cada caixa por padrão é criada apenas com o conteúdo, a única exceção é o body que é criado com uma margin de 8px. Podemos acrescentar em cada elemento, valores para a margin, padding, border e outline.

Propriedades das caixas

  • Conteúdo

    • Todo conteúdo (content) possui um tamaho de caixa chamado box size composto de altura (height) e largura (width).

  • Panding

    • Fora da área de conteúdo temos a área de preenchimento (padding). O padding de fica da borda para dentro (border).

  • Border

    • É a ária da borda do conteúdo. Cada borda pode ser configurada com uma cor, espessura e formato.

  • Outline

    • O espaço entre a margem e a borda onde podemos determinar o contorno (outline) é muito pouco utilizado. Ele é um traçado visual que podemos criar fora da borda e o cálculo da sua espessura faz parte da margem estabelecida.

  • Margin

    • A margem é a parte exterior que fica depois da borda.

A nova propriedade box-sizing define as dimensões height e width a partir da borda (border-box) e não apenas a partir do conteúdo (content-box).

A medida total da height e width que essa caixa vai ocupar é feita pela soma de todas as medidas, margin, padding, border e content.

Box inline e box em bloco

Partindo do princípio de que todo conteúdo em CSS é representado por caixas temos dois tipos de caixas.

O sorthand das medidas do border e do outline são:

  • border: tamanho tipo cor;
  • outline: tamanho tipo cor;

A margin e o padding são configuradas com um valor para os quatro lados, um valore para direita e esquerda, outro para em cima e em baixo, e também um valor para cada lado em centido horário, cima, direita, baixo, esquerda.

Outra forma de configuração é colocando auto e deixando com que o navegador caucule as margens deixando o bloco sempre no meio.

Box inline

O box inline é usado quando queremos trabalhar com uma caixa dentro do conteúdo. Sua caracteristica é ocupar somente o espaço que lhe cabe sem precisar quebrar ou pular linha.

Existem algumas tags in block que talves você já conheça.

  • <Addres> pode ser usado em uma variedade de contextos, como fornecer informação de contato de uma empresa no header da página, ou indicar o autor de um artigo incluindo um elemento address dentro do article.

  • <Article> Este poderia ser o post de um fórum, um artigo de revista ou jornal, um post de um blog, um comentário enviado por um usuário, um gadget ou widget interativos, ou qualquer outra forma de conteúdo independente.

  • <aside> para adicionar um conteúdo segundário ou que tenha a ver com o contepudo principal. Elas muitas vezes contem explicações laterais, como a definição de um glossário; conteúdo vagamente relacionado, como avisos; a biografia do autor; ou, em aplicações web, informações de perfil ou links de blogs relacionados.

  • <blockquote> indica que o texto incluído é uma longa citação

  • <canvas> pode ser utilizado para desenhar gráficos utilizando scripts, além de desenhar gráficos, ele pode ser usado para fazer composições de fotos e também para animações

  • <dd> fornece detalhes ou uma definição mais completa do termo definido por dt numa lista de descrições dl.

  • <dl> engloba uma lista de pares de termos e descrições. Um uso comum para este elemento é para implementar um glossário ou exibir metadados(uma lista de pares chave e valor).

  • <dt> identifica um termo na lista de definição. Este elemento pode ocorrer somente em um elemento filho de
    . Geralmente seguido por um elemento dd.

  • <div> é um container genérico para conteúdo de fluxo, que de certa forma não representa nada. Ele pode ser utilizado para agrupar elementos para fins de estilos (usando class ou id), ou porque eles compartilham valores de atributos, como lang. Ele deve ser utilizado somente quando não tiver outro elemento de semântica tal como article ou nav.

  • <fieldset> é usado para agrupar elementos, assim como labels dentro de um formulário web.

  • <figcaption> representa uma legenda ou uma legenda associada com uma figura ou ilustração descrita pelo resto dos dados do elemento

  • <figure> representa o conteúdo independente, frequentemente com uma legenda (

  • <footer> representa um rodapé para o seu sectioning content (conteúdo de seção) mais próximo ou sectioning root elemento (ou seja, seu parente mais próximo, article, aside, nav, section, blockquote, body, details, fieldset, figure, td. Normalmente um rodapé contém informações sobre o autor da seção de dados, direitos autorais ou links para documentos relacionados.

  • <form> representa uma seção de um documento que contém controles interativos que permitem ao usuário submeter informação a um determinado servidor web.

  • <h1-h6> representam seis níveis de título de seção. h1 É o nível de seção mais alto e h6 é o mais baixo.

  • <header> representa um grupo de suporte introdutório ou navegacional. Pode conter alguns elementos de cabeçalho mas também outros elementos como um logo, seções de cabeçalho, formulário de pesquisa, e outros.

  • <hr> representa uma quebra temática entre elementos de nível de parágrafo (por exemplo , uma mudança da cena de uma história, ou uma mudança de tema com uma seção). Nas versões anteriores do HTML, representava uma linha horizontal. Pode continuar sendo exibida como uma linha horizontal nos navegadores, mas agora está definida em termos semânticos, em vez de termos de apresentação.

  • <main> define o conteúdo principal dentro do em seu documento ou aplicação. Entende-se como conteúdo principal aquele relacionado diretamente com o tópico central da página ou com a funcionalidade central da aplicação.

  • <nav> para o navegador ou menu. Representa uma seção de uma página que aponta para outras páginas ou para outras áreas da página, ou seja, uma seção com links de navegação.

  • <noscript> define uma seção de html a ser inserida se um tipo de script não é suportado pela página ou se o script está desativado no navegador.

  • <ol> para criar uma lista organizada com hierarquia.

  • <p> para criar paragrafo.

  • <pre> é a tag utilizada para representar texto pré-formatado. Um texto dentro desse elemento é tipicamente exibido em uma fonte não proporcional da mesma maneira em que o texto original foi disposto no arquivo. Espaços em branco são mantidos no texto da mesma forma em que este foi digitado.

  • <section> representa uma seção genérica contida em um documento HTML, geralmente com um título, quando não existir um elemento semântico mais específico para representá-lo.

  • <table> para a criação de tabelas. Representa dados em duas dimensões ou mais.

  • <tfoot> é um elemento HTML que define um conjunto de linhas as quais farão parte do rodapé de uma tabela HTML

  • <video> O elemento HTML video é utilizado para incorporar conteúdo de vídeo em um documento HTML ou XHTML.

Na tag "Style" dentro da head ou na folha de CSS podemos alterar as caixas dos conteúdos. Para transformar um box inline em um box em bloco basta adicionar o atributo display: inline-block;, também podemos configurar o bloco como inline.

Uma dica é usar o Dev tools do navegador para alterar a pagina e mostrar em tempo real para o cliente como as mudanças ficam no site, caso o cliente goste do resultado é só implementar no código original.
